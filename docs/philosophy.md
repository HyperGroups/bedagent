# bedagent 哲学

## 目标用户

bedagent 的目标用户不是“更高效的工程师”，而是：

> 一个没有手脚、懒到极致、躺在床上的思想者。

他不想操作电脑，不想管理工具，不想盯流程；但他一直在想。

所以 bedagent 的任务不是简单“帮他偷懒”，而是把低操作状态下的思考变成现实结果。

## sofagent 上下文

bedagent 与 sofagent 属于同一个“躺着让 Agent 干活”的语义家族。

- sofagent：sofa + agent，躺在沙发上，让 Agent 把活干了；
- bedagent：bed + agent，躺在床上，把想法交给 Agent 去推演和执行。

sofagent 是 bedagent 的灵感来源和上下文来源，尤其是“Agent 需要纪律”这条主线。

详见 [sofagent-context.md](sofagent-context.md)。

## sofagent vs bedagent

| 维度 | sofagent | bedagent |
|------|----------|----------|
| 第一性问题 | Agent 不守规矩怎么办 | 人懒得操作但一直在想怎么办 |
| 默认姿态 | 用户坐在电脑前监督 Agent | 用户躺在床上，用语音/短命令指挥 |
| 核心对象 | Agent 行为 | 人的想法到 Agent 行动 |
| 成功标准 | Agent 少犯错 | 用户少操心，想法不流失，结果可控 |
| 产品隐喻 | 纪律委员 | 思想者外骨骼 |

sofagent 管的是：

> Agent 怎么干。

bedagent 管的是：

> 人怎么想，Agent 怎么把想法变成结果。

## 第一原则

**懒到极致的人是一等公民。**

这意味着：

1. 不能默认用户愿意打字；
2. 不能默认用户愿意读长报告；
3. 不能默认用户愿意看 diff；
4. 不能默认用户愿意管理分支、脚本、任务队列；
5. 不能默认用户随时坐在屏幕前；
6. 不能默认用户愿意在低价值分支上无限消耗注意力。

但懒不等于放弃控制。bedagent 的设计目标是：

> 少打扰，但关键时刻必须叫醒人。

懒也不是单一原则。bedagent 区分四种懒：

| 类型 | bedagent 态度 |
|------|---------------|
| 行动懒 | 保护：机器能做的操作不要让人做 |
| 表达懒 | 补足：系统负责复述、归类、追问、结构化 |
| 思考懒 | 约束：不能替人逃避目标、风险、取舍和责任 |
| 认知懒 | 强化：帮人剪掉低价值分支，保护主线注意力 |

所以更精确的原则是：

> 保护行动懒，补足表达懒，约束思考懒，强化认知懒。

## 第二原则：最智者

懒人哲学只告诉我们“不要浪费”，但它还需要一个判断者来决定“不浪费在哪里”。

这个角色叫 **最智者**。

最智者不是执行 Agent，而是 bedagent 大脑里的最高判断层：

- 它保护主线；
- 它剪掉低价值分支；
- 它只问关键问题；
- 它把长推演压成短结论；
- 它在不值得执行时说“先别动”。

一句话：

> 懒人哲学给方向，最智者做取舍，Agent 手脚去执行。

详见 [lazy-philosophy-and-sage.md](lazy-philosophy-and-sage.md)。

## 思想优先于执行

人在床上最多做的事情是想。

这些想法往往是碎片化的：

- “这个方向不太对。”
- “Agent 乱想是不是也要沙盒？”
- “明天要把这个做成文档。”
- “这个项目是不是该换一个哲学？”

bedagent 首先要接住这些想法，而不是马上执行。

但快速思考也会制造无限分支。bedagent 不能让每个念头都展开成一个项目。它应该维护当前主线，判断新想法是展开、暂存、合并，还是剪掉。

bedagent 的节奏：

> 想的时候快，剪枝要狠，动手前慢，执行要稳。

## Brain before Hands

Agent 有两种失控：

| 类型 | 问题 | bedagent 对策 |
|------|------|---------------|
| 乱想 | 没想清楚就下结论 | 推演沙盒 / Agent Brain |
| 乱改 | 没验证就改真实世界 | 执行沙盒 / Agent Hands |

bedagent 的核心原则：

> 先让 Agent 在脑子里犯错，再让它在沙盒里试错，最后才允许碰真实世界。

## 一句话

> 我躺着想，Agent 起来干。

更完整一点：

> bedagent 是思想者的 Agent 外骨骼：接住想法，陪他推演，替他执行，并在危险前停下。
