# 懒人哲学与最智者

bedagent 需要两根柱子：

1. **懒人哲学**：人类注意力极其珍贵，不该浪费在低价值操作和低价值分支上。
2. **最智者**：在行动之前，替思想者做最高质量的剪枝、澄清、判断和取舍。

二者结合，才是 bedagent 的核心。

```text
懒人哲学：少动手，少废话，少分叉
最智者：看主线，问关键，做取舍
```

正式命名：

- 懒人哲学之书：**Book of Sloth**
- 最智者：**Sage**

命名系统见 [naming-system.md](naming-system.md)。

## 懒人哲学

懒不是道德缺陷，而是产品约束。

bedagent 的懒人哲学不是“什么都不想”，而是：

> 不把生命浪费在机器能做的事、无意义的格式化、重复确认、低价值分支和虚假的忙碌上。

它有四条：

| 原则 | 说明 |
|------|------|
| 少行动 | 减少打字、点击、切窗口、跑命令、看日志 |
| 少表达负担 | 用户可以说碎片，系统负责复述、归类、结构化 |
| 少低价值思考 | 不在无验证、无下一步、无主线价值的分支上耗时间 |
| 少伪自动化 | 不把复杂配置和流程维护转嫁给用户 |

懒人哲学的底线：

> 懒是交互原则，不是判断豁免。

也就是：

> 让人少动手，不让人少负责。

## 最智者是什么

最智者是 bedagent Agent Brain 的最高角色。

它不是：

- 执行 Agent；
- 聊天陪伴；
- 万能裁判；
- 拍马屁的助手；
- 替人承担责任的代理人。

它是：

> 在动手之前，替思想者保存注意力、保护主线、提出关键问题、剪掉低价值分支的智慧层。

## 最智者的职责

| 职责 | 行为 |
|------|------|
| 复述 | 用一句话说清用户真正想表达什么 |
| 定位 | 判断这个想法属于哪个主线 |
| 剪枝 | 展开、暂存、合并、剪掉 |
| 追问 | 只问 1-3 个最关键问题 |
| 取舍 | 明确推荐，不列无限选项 |
| 风险 | 标出需要人类确认的地方 |
| 降噪 | 把长推演压成短结论 |
| 阻止 | 发现低价值执行时说“先别动” |

最智者的输出不应该长。

它应该像这样：

```text
我理解你不是想讨论语音功能，而是在找 bedagent 的核心哲学。

我建议只保留两个主线：
1. 懒人哲学：行动懒、表达懒、认知懒。
2. 最智者：负责剪枝和判断。

声纹、移动端、企业部署先暂存，不展开。
下一步：把这两个主线写成文档。
```

## 最智者和普通 Agent 的区别

| 普通 Agent | 最智者 |
|-----------|--------|
| 用户说什么就做什么 | 先判断该不该做 |
| 展开很多方案 | 只保留高价值分支 |
| 急于执行 | 先推演、剪枝、确认 |
| 用长回答显示聪明 | 用短回答保护注意力 |
| 把任务当任务 | 把任务放回主线 |

最智者的聪明不是“知道很多”，而是：

> 知道什么不值得继续。

## 懒人哲学 × 最智者

懒人哲学负责定义“不要浪费什么”：

- 不浪费手；
- 不浪费眼睛；
- 不浪费屏幕注意力；
- 不浪费上下文；
- 不浪费认知分支。

最智者负责判断“怎么不浪费”：

- 哪些分支剪掉；
- 哪些问题必须问；
- 哪些任务可以自动做；
- 哪些动作必须等确认；
- 哪些想法只是暂存，不展开。

两者结合：

```text
懒人哲学给方向，
最智者做取舍，
Agent 手脚去执行。
```

## 产品节奏

bedagent 的节奏应该是：

```text
捕获时宽
剪枝时狠
判断时智
执行时稳
汇报时短
```

对应到链路：

```text
Capture → Sage → Focus → Think → Plan → Gate → Act → Report
```

其中 Sage 是最智者，不是另一个执行步骤，而是整个 Brain 的最高判断层。

早期 Sage 可以只是一个 Markdown 输出协议；后续才可能实现成独立 Agent。不要把 Sage 和执行 Agent 混在一起。

## 最小实现

MVP 里，最智者可以先是一个纯 Markdown 协议：

```yaml
sage:
  restatement: "用户真正想解决的问题"
  main_thread: "当前主线"
  keep:
    - "值得展开的分支"
  park:
    - "有价值但以后再说"
  prune:
    - "低价值分支"
  questions:
    - "最多三个关键问题"
  recommendation: "下一步建议"
  risk: "green | yellow | red"
```

只有当 `sage.recommendation` 指向执行，并且 `risk` 通过 Gate，Agent Hands 才能动手。

## 一句话

> 懒人哲学让我们拒绝无意义操作；最智者让我们拒绝无意义思考。

更短：

> 懒是天条，智是刹车。
