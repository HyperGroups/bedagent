# 语音控制规划

## 场景

人在床上，不适合长时间看屏幕，但脑子还在想。

bedagent 的语音能力不是“聊天”，而是：

> 低屏幕思想捕获与任务遥控。

## 设计原则

| 原则 | 说明 |
|------|------|
| 懒人优先 | 默认少输入、少看屏、少维护 |
| 短反馈 | 语音反馈默认 10-20 秒，只播报结论、风险、下一步 |
| 可打断 | “暂停 / 等等 / 停一下”必须立即生效 |
| 不猜测 | STT 置信度低时追问，不执行 |
| 不读秘密 | 密钥、密码、隐私不通过 TTS 读出 |
| 高风险确认 | 红色动作不能只靠一句含糊语音批准 |

## 基础口令

| 口令 | 行为 |
|------|------|
| “记一下” | 捕获想法，不执行 |
| “帮我想想” | 进入推演沙盒 |
| “整理成任务” | 生成计划和验收标准 |
| “问我三个问题” | 澄清想法 |
| “可以执行” | 进入风险闸门 |
| “暂停” | 停止下一步 |
| “继续” | 从 checkpoint 恢复 |
| “取消” | 终止任务 |
| “汇报一下” | 短摘要播报 |

## 技术栈候选

| 项目/方向 | 可借鉴点 |
|-----------|----------|
| Pipecat | 实时语音/多模态 Agent 管线，STT/TTS 可插拔 |
| TEN Framework | VAD、Turn Detection、低延迟实时会话 |
| Kiwi Voice | OpenClaw 语音接口、唤醒词、barge-in、TTS |
| openWakeWord + Whisper + Piper/Kokoro | 本地优先语音栈 |
| 系统级听写 | MVP 零集成入口 |

## 分阶段

### P0：不接麦克风，先定义契约

- 定义 voice intent；
- 定义短反馈格式；
- 定义风险确认话术；
- 用系统听写手动输入测试。

### P1：OpenClaw 语音适配

- 优先参考 Kiwi Voice；
- 支持 push-to-talk；
- 支持任务开始/等待确认/完成/失败四类 TTS。

### P2：跨平台语音网关

- 评估 Pipecat / TEN；
- 手机或旧电脑作为床边遥控器；
- 主机继续运行 Agent。

## 不做什么

- 不做通用语音助手；
- 不做情感陪伴；
- 不朗读长 diff；
- 不允许语音单独批准高风险动作；
- 早期不自研 STT/TTS 模型。
