# 懒与剪枝原则

“懒到极致的人是一等公民”不能被理解成“什么都不想，什么都交给 Agent”。bedagent 要服务的是低操作状态下的高质量思考。

所以懒需要分型。

剪枝这件事由“最智者”承担：懒人哲学定义什么不值得浪费，最智者负责判断哪些分支应该展开、暂存、合并或剪掉。详见 [lazy-philosophy-and-sage.md](lazy-philosophy-and-sage.md)。

## 四种懒

| 类型 | 含义 | bedagent 态度 |
|------|------|---------------|
| 行动懒 | 不想打字、点击、切窗口、跑命令、读日志 | 保护。机器能做的操作不要让人做 |
| 表达懒 | 想法很多，但不想整理成完整 prompt | 补足。系统负责复述、归类、追问、结构化 |
| 思考懒 | 不想面对目标、风险、取舍和责任 | 约束。系统可以降低负担，但不能替人逃避判断 |
| 认知懒 | 不想在低价值分支上浪费注意力 | 保护。系统要帮人剪枝、合并、聚焦主线 |

一句话：

> 保护行动懒，补足表达懒，约束思考懒，强化认知懒。

## 懒不是判断豁免

bedagent 的懒原则不是：

> 用户懒，所以 Agent 可以替用户做所有判断。

而是：

> 用户懒，所以系统要减少无价值操作；但关键判断仍要留给人。

更短：

> 让人少动手，不让人少负责。

## 快速思考的矛盾

人在床上会快速思考，想法跳得快。如果系统不断追问，会打断思路；如果系统放任发散，又会进入无限分支。

所以 bedagent 要区分两个阶段：

```text
想的时候快
动手前慢
```

还要补上剪枝：

```text
想的时候快，
剪枝要狠，
动手前慢，
执行要稳。
```

## 剪枝模式

快速思考不是每个分支都展开。bedagent 应该默认维护一条“当前主线”，对新想法做四种处理：

| 处理 | 说明 | 示例 |
|------|------|------|
| 展开 | 与当前主线强相关，值得继续想 | Agent 大脑 / 沙盒系统 |
| 暂存 | 有价值但不是当前主线 | 声纹识别、手机端遥控 |
| 合并 | 与已有主题重复或相近 | “语音确认”合并到 Voice Gate |
| 剪掉 | 低价值、不可验证、只是绕圈 | 纯概念命名争论 |

输出应该很短：

```text
我建议只展开 2 个分支：
1. Agent 大脑：主线，继续。
2. 执行沙盒：主线，继续。

语音声纹先放 backlog；命名争论先剪掉。
```

## 主线保护

bedagent 需要随时知道当前主线是什么。

示例：

```yaml
focus:
  main_thread: "设计 bedagent 的哲学与 MVP"
  active_questions:
    - "思想者需要什么"
    - "Agent 大脑如何防乱想"
    - "执行沙盒如何防乱改"
  parking_lot:
    - "声纹识别"
    - "移动端遥控器"
    - "企业部署"
```

任何新想法先被放到这个坐标系里，而不是无条件展开。

## bedagent 的认知闸门

在推演沙盒里，Agent Brain 不只生成方案，还要做剪枝判断：

1. 这个分支是否服务当前主线？
2. 是否有下一步动作？
3. 是否可验证？
4. 是否重复已有主题？
5. 现在展开是否会拖慢主线？

如果答案不好，先暂存或剪掉。

## 模式设计

| 模式 | 目标 | 行为 |
|------|------|------|
| Capture | 不打断思路 | 只记，不问，不执行 |
| Focus | 保护主线 | 归类、合并、剪枝 |
| Think | 深入推演 | 只展开少数高价值分支 |
| Plan | 准备执行 | 变慢，列风险和验收 |
| Act | 稳定执行 | 沙盒、验证、短汇报 |

## 最终原则

> 懒不是不想，而是不愿意在低价值分支上想。

bedagent 要让思想者保留快速思考的自由，同时不被无限分支吞掉。
