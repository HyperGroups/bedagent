# bedagent 能力地图

bedagent 的功能从“没有手脚的思想者需要什么”出发，而不是从“Agent 框架能做什么”出发。

## 五大模块

```text
Mind   捕获与思考
Voice  低屏幕交互
Gate   风险与纪律
Hands  执行沙盒
Memory 记忆与审计
```

## A. Mind 捕获与思考

人在床上的输入通常不是完整任务，而是碎片想法。

功能：

- 想法捕获：“记一下”；
- 自动归类：产品、技术、待办、风险、反思；
- Focus 剪枝：展开、暂存、合并、剪掉；
- 推演沙盒：复述、澄清、列方案、找风险；
- 任务整理：把“我觉得”变成“下一步做什么”；
- 反问模式：“问我三个问题，把这事问清楚”。

核心价值：

> 不让想法流失，也不让低价值分支吞掉注意力。

Focus 剪枝的四种动作：

| 动作 | 含义 |
|------|------|
| 展开 | 与主线强相关，值得继续想 |
| 暂存 | 有价值但不是现在 |
| 合并 | 与已有主题重复或相近 |
| 剪掉 | 低价值、不可验证、只是绕圈 |

## B. Voice 低屏幕交互

床上模式必须默认少看屏幕。

功能：

- 语音输入；
- 短口令：暂停、继续、取消、汇报；
- TTS 短反馈；
- 夜间安静模式；
- 低置信度追问，不猜测执行。

核心价值：

> 不看屏幕也能推进，但不牺牲安全。

## C. Gate 风险与纪律

懒人可以懒，Agent 不能乱。

功能：

- 绿/黄/红风险分级；
- action manifest；
- 高风险确认；
- 幂等检查；
- 纪律规则：先读后写、验证再干、谨慎修改。

核心价值：

> 少打扰，但关键动作必须停下来问。

## D. Hands 执行沙盒

Agent 是思想者的手脚，但手脚不能乱动。

功能：

- 临时分支 / git worktree；
- sandbox repo；
- dry-run；
- 测试验证；
- diff 摘要；
- 成功后再请求合并。

核心价值：

> 让 Agent 在沙盒里试错，不要在真实世界试错。

## E. Memory 记忆与审计

bedagent 要越用越懂用户怎么想。

功能：

- 反思记忆；
- 用户偏好；
- task logs；
- 审计报告；
- 短结果汇报；
- 失败经验沉淀。

核心价值：

> Agent 不能自证清白，系统要保留证据。

## 最小闭环

MVP 不应该先做复杂语音栈，而应该先跑通：

```text
Capture → Focus → Think → Plan → Confirm → Act Sandbox → Short Report
```

也就是：

1. 记录一个想法；
2. 先剪枝，保护主线；
3. 在脑内沙盒推演；
4. 整理成任务；
5. 用户确认；
6. 在执行沙盒里动手；
7. 用一句话汇报结果。

语音是入口，沙盒是手，大脑是核心。
